1. 사양
서버는 1분 간 암호화된 공격 대상 또는 복호화 키를 제공
간격은 1시간


2. 데이터 길이
플래그: 2 비트
{
    00: 정보 없음
    01: 공격 정보
    10: 복호화 키
    11: 전체 취소
}

공격 대상 정보
IP 주소: 4 바이트 (IPv4)
포트 주소: 2 바이트
공격 시작 년도: 6 비트 ( 0~255 + 현재 년도)
공격 시작 날짜: 9 비트 (1~365 일)
공격 시작 시간: 5 비트 (시간 단위)
공격 길이: 5 비트 (시간 단위)

암호화 방식
ECC 비대칭키 암호화 방식 
복호화 키: 32 바이트

혹은 비대칭키를 서명용으로만 사용, 대칭키로 암호화는 하이브리드 암호 사용


3. 정보 배포
암호화 또는 난독화 한 정보를 config.json, logo.png 등의 파일로 위장하여 작성
일정 시간마다 서버 프로그램이 이를 읽어 배포


4. cli 기능
(1) 시간 확인 - 완료
여러 시간대 포함 - 완료

(2) 상태 확인 - 완료
서버 구동 상태 - 완료
지난 배포 봇 갯수 확인 - 완료
현재 패킷 데이터 상태 - 완료

(3) 패킷 데이터 확인 - 완료
현재 탑재된 패킷 데이터 확인 - 완료

(4) 패킷 데이터 작성 - 완료
플래그 작성 - 완료
이에 따른 데이터 작성 - 완료

5. 스케쥴러 기능
(1) 매 정각 패킷 broadcast - 완료
패킷 형태 조정(바이트 단위) - 완료

(2) 매 1분 패킷 broadcast 정지 및 패킷 삭제 - 완료
패킷 삭제 - 완료

6. 암호화 기능
(1) 공개키를 이용한 서명
서버 공개키 제공

(2) 대칭키를 이용한 패킷 암호화
대칭키 생성




# 암호화 추가 이후
패킷 내용 바이트 화 -> 암호화 -> 전달
대칭 키 전달 

복호화 -> 파싱
